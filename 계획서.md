## 1. 개요
==============================

**프로젝트 목표**: AI 에이전트 선택 및 제어 시스템
**유형**: 자가 개선형 실행 엔진 (다중 에이전트 경쟁 방식)
**현재 레벨**: 레벨 3 구현됨 (노이즈가 있지만 사용 가능한 그래프)

**핵심 철학**: "AI를 만드는 것이 아니라, AI를 선택하고 제어하는 시스템"

## 2. 완전한 아키텍처 설계
==============================

### 2.1 현재 구현 (레벨 3) {#21-현재-구현-레벨-3}
```
사용자 입력
   |
   v
러너 (core/runner.py) - 오케스트레이터
   |
   v
에이전트 (core/agents.py) - 정책 실행
   |  - refine: "정리하고 깔끔하게"
   |  - balance: "균형 잡힌 접근"
   |  - expand: "상세하고 확장된"
   |
   v
노드 (core/node.py) - AI 처리
   |
   v
평가자 (core/evaluator.py) - 점수 평가
   |
   v
승자 선택 - 최고 정책 결과
   |
   v
메모리 (core/memory.py) - 학습 저장
   |
   v
출력
```

   |  - 성공 추적
   |  - 정책 적응

   |  - 런타임 패칭

   |  - 구조 적응

   v

## 4. 구성 요소 사양
==============================

### 4.1 핵심 구성 요소 {#41-핵심-구성-요소}

#### 러너 (core/runner.py)
**목적**: 메인 오케스트레이터 및 흐름 제어
**책임**:
- 정책 선택 및 실행
- 재시도 로직 (최대 3회 시도)
- 메모리 통합
- 자가 수정 트리거링
**인터페이스**: `run_node(fn, input_data, threshold=8, max_retry=3)`

#### 에이전트 (core/agents.py)
**목적**: 다중 에이전트 경쟁 시스템
**정책**:
- **refine**: "정리하고 깔끔하게" 접근
- **balance**: "균형 잡히고 절제된" 접근
- **expand**: "상세하고 확장된" 접근
**진화**: 성공 패턴 기반 동적 정책 생성

#### 노드 (core/node.py)
**목적**: AI 처리 실행
**기능**: `ai_call()` 추상화
**제공자**: OpenAI, 로컬 AI, 모의 폴백

#### 평가자 (core/evaluator.py)
**목적**: 결과 품질 평가
**점수화 요소**:
- 기본 점수 (5점)
- 길이 보너스 (최대 3점)
- 정책별 보너스
- 범위: 1-10점

### 4.2 고급 구성 요소 {#42-고급-구성-요소}

#### 의도 레이어 (core/intent.py) - [구현 예정]
**목적**: 입력 의미 분석
**기능**:
- 입력 분류 (쿼리, 명령, 분석)
- 컨텍스트 추출
- 목표 식별
- 우선순위 할당

#### 툴 레이어 (core/tools/) - [구현 예정]
**목적**: 외부 시스템 상호작용
**툴**:
- 파일 작업 (읽기, 쓰기, 목록)
- 로그 분석 (파싱, 필터링, 요약)
- 시스템 상호작용 (프로세스, 메모리, 네트워크)
- 웹 작업 (스크래핑, API 호출)

#### 메모리 시스템 (core/memory.py)
**목적**: 학습 및 패턴 저장
**기능**:
- 성공 패턴 저장
- 정책 효과 추적
- 과거 컨텍스트 유지
- 학습 피드백 통합

## 6. 기술 사양
==============================

### 6.1 성능 요구사항 {#61-성능-요구사항}
- **응답 시간**: 단순 쿼리 < 5초
- **동시성**: 여러 동시 실행 지원
- **메모리**: TTL과 효율적 캐싱
- **신뢰성**: 폴백 메커니즘으로 99.9% 가동 시간

- **시스템 API**: 프로세스 관리, 모니터링
- **웹 서비스**: 외부 API 호출

### 6.3 보안 고려사항 {#63-보안-고려사항}

- **파일 접근**: 샌드박스 작업
