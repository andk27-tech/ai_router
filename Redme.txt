# AI Router Project

## 개요

이 프로젝트는 Python AST 기반으로 함수 호출 관계를 추출하고, 이를 그래프로 구성한 뒤 라우팅/스코어링까지 이어가는 실험용 AI 라우터 시스템이다.

현재 단계는 **Level 3 (noisy but usable graph)** 까지 구현된 상태다.

---

## 현재 구현 단계

* Level 1: AST extraction ✔
* Level 2: graph construction ✔
* Level 3: noisy but usable graph ✔
* Level 4: semantic dependency graph ❌ (미구현)
* Level 5: AI router graph ❌ (미구현)

---

## 프로젝트 구조

```
ai_router/
│
├── core/
│   ├── graph_v5.py        # AST 기반 함수 호출 그래프 생성 (핵심)
│   ├── graph_clean.py     # 그래프 노이즈 제거
│   ├── router_rank.py     # 노드 스코어링 및 정렬
│   ├── score.py           # 개별 노드 점수 계산
│   └── router_run.py      # 전체 파이프라인 실행 엔트리
│
├── venv/                  # 가상환경
└── ...
```

---

## 핵심 파일 설명

### 1. graph_v5.py

* AST를 분석하여 함수 간 호출 관계를 추출
* execution DAG 또는 call graph 생성
* 현재 시스템의 핵심 데이터 소스

---

### 2. graph_clean.py

* 그래프에서 불필요한 노드 제거
* 파일 경로, 일부 외부 라이브러리 제거

현재 로직:

* "./" 포함 경로 제거
* "/" 포함 키 제거 (과도한 필터링 가능)
* json, FastAPI 같은 노드 제거

⚠️ 현재 Level 3 상태의 주요 bottleneck

---

### 3. router_rank.py

* score_node 기반으로 각 호출 대상 노드 점수 계산
* 점수 기준으로 정렬

```python
items = [(node, score_node(node))]
items.sort(reverse=True)
```

---

### 4. score.py

* 노드 중요도 계산 로직
* 현재는 단순 heuristic 기반

---

### 5. router_run.py

* 전체 pipeline 실행

실행 흐름:

```python
from core.graph_v5 import build_execution_dag
from core.graph_clean import clean_graph
from core.router_rank import rank_graph


g = build_execution_dag()
g = clean_graph(g)
g = rank_graph(g)
print(g)
```

## 현재 문제점 (중요)

### 1. 과도한 noise filtering

* '/' 제거로 인해 그래프 손실 가능

### 2. semantic resolution 없음

* 단순 문자열 기반 node 연결
* cross-file function resolution 없음

### 3. dependency graph 미완성

* 실제 execution dependency가 아니라 call list 수준
